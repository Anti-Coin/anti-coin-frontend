# VERIFY.md
Authoritative for: frontend acceptance evidence.

## Done Means Verified
Completion requires at least one of:
- automated tests
- manual verification steps
- runtime evidence (screens/logs/metrics)

## Required Coverage
1) Functional correctness
2) Policy correctness (`serve_allowed`, status gating, visibility)
3) Failure/degradation behavior
4) Observability
5) Non-regression
