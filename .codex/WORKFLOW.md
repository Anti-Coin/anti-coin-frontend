# WORKFLOW.md
Authoritative for: frontend execution sequence.

## Default Flow
1) Analyze
2) Discuss/Critique
3) Test Design
4) Patch
5) Verify

## Stage Requirements
### Analyze
- Restate objective.
- Identify contract and policy constraints.
- List missing variables.

### Discuss/Critique
- Challenge assumptions.
- Identify failure modes and edge cases.
- Choose lowest-risk path.

### Test Design
- Define failure modes first.
- Plan contract tests and UI-state tests.

### Patch
- Keep changes minimal and scoped.
- Avoid speculative redesign.

### Verify
- Provide evidence and residual risks.
